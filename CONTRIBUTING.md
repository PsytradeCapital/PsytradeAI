# Contributing to PsyTradeAI

We welcome contributions to PsyTradeAI! This document provides guidelines for contributing to the project.

## ğŸ¤ How to Contribute

### 1. Fork the Repository
- Fork the PsyTradeAI repository to your GitHub account
- Clone your fork locally
- Create a new branch for your feature or bug fix

### 2. Development Setup
```bash
git clone https://github.com/your-username/PsyTradeAI.git
cd PsyTradeAI
```

### 3. Making Changes
- Follow the existing code style and conventions
- Add comprehensive comments to your code
- Ensure all functions are properly documented
- Test your changes thoroughly

### 4. Code Standards

#### MQL5 Coding Standards
- Use descriptive variable and function names
- Follow camelCase for variables and PascalCase for classes
- Add comprehensive comments for complex logic
- Use proper error handling
- Optimize for performance

#### Documentation Standards
- Update relevant documentation for any changes
- Include code examples where appropriate
- Maintain consistency with existing documentation style

### 5. Testing Requirements
- Test all new features thoroughly
- Ensure backward compatibility
- Run backtests to verify performance
- Test on demo accounts before submitting

### 6. Submitting Changes
1. Commit your changes with clear, descriptive messages
2. Push to your fork
3. Create a Pull Request with:
   - Clear description of changes
   - Rationale for the changes
   - Test results and screenshots if applicable
   - Any breaking changes noted
